# Vision

Vi bygger ét system der virker.

**Grundighed.** Vi forstår før vi handler. Symptomer er ikke årsager. Evidens, ikke antagelser.

**Metode.** Flow: forstå, validér, byg, verificér. Hvert skridt bekræftet før det næste.

**Princip.** Én sandhed på tværs af systemet. Logik findes ét sted. UI viser hvad systemet gør.

**Enkelhed uden kompromis.** Det enkleste der løser problemet. Kompleksitet uden værdi er gæld.

**Optimering, ikke lapper.** Hver løsning gør systemet bedre. Find rod-årsagen. Når du rører noget, efterlad det stærkere end du fandt det.

**Mål.** Stork skal kunne forstås, vedligeholdes og vokse — af to partnere med AI som arbejdsredskab.

# Sådan arbejder vi sammen

Fem aktører: Mathias, Claude.ai, Claude Code, Lovable, Codex.

**Mathias** beslutter, definerer retning.
**Claude.ai** er mellemmand, formulerer prompts, holder flow.
**Claude Code, Lovable og Codex** hjælper hinanden med at vi sammen får et system som overgår alle forventninger.

**Fælles grundlag før session.** Arbejd mod samme sandhed. `origin/main` er kilden. Auto-regenererede snapshot-filer i `docs/` (system, UI, krydsreferencer) er fælles teknisk reference for alle bygger-AI'er.

Hver rapport indeholder commit-hash, ændrede filer, bekræftelse på push.

Diagnose før plan. Symptom + evidens + rod-årsag + fix. Aldrig plan uden evidens.

Én ordre = én commit. Bundling er afvist.

Ved konflikt mellem AI'er: ord-for-ord citater med linjenumre. Den med citat vinder.

# Sådan hænger data sammen

Tre principper for data:

- **Én sandhed.** Hver data-type har ét sted den lever. Alt andet er views.
- **Styr på data.** GDPR-compliance er ikke add-on. Persondata håndteres efter retention og samtykke.
- **Sammenkobling.** Data er forbundet på tværs — ingen silo'er.

Data tilhører dimensioner. Listen er ikke udtømmende — det er en måde at tænke på.

- **Medarbejder:** vagter, salg, jobansøgning, løn, rettigheder, annulleringer, pulsmåling, spil, anciennitet
- **Leder:** team, kunder
- **Ejere:** hele systemet, strategiske beslutninger

Adgang har to dimensioner: rolle og team. Rolle bestemmer hvilke dele af systemet en bruger må se. Team bestemmer hvilken data inden for det.
